# คู่มือการใช้งานโหมด Offline

## ความสามารถในโหมด Offline

แอปพลิเคชันนี้รองรับการทำงานแบบ offline อย่างเต็มรูปแบบ:

### ✅ ฟีเจอร์ที่ใช้งานได้ใน Offline Mode

1. **บันทึกข้อมูลป้ายทะเบียน**
   - สแกนภาพถ่ายที่บันทึกไว้ (ไม่สามารถใช้ OCR ออนไลน์)
   - กรอกข้อมูลป้ายทะเบียนด้วยตนเอง
   - บันทึก GPS location
   - เก็บภาพถ่ายในเครื่อง

2. **ดูข้อมูลที่บันทึกไว้**
   - ดูประวัติการสแกนทั้งหมด
   - ค้นหาข้อมูลที่บันทึกไว้
   - ตรวจสอบสถานะการซิงค์

3. **การจัดการข้อมูล**
   - Export ข้อมูลเป็นไฟล์ JSON
   - ล้างแคชข้อมูล
   - ดูสถิติการใช้งาน

### ⚠️ ฟีเจอร์ที่ต้องใช้อินเทอร์เน็ต

1. **OCR อัตโนมัติ** - ต้องเชื่อมต่ออินเทอร์เน็ตเพื่อใช้ API
2. **ซิงค์ข้อมูลกับเซิร์ฟเวอร์** - ต้องออนไลน์เพื่อส่งข้อมูล
3. **ดึงข้อมูลการลงทะเบียน** - ต้องออนไลน์เพื่ออัพเดทข้อมูล

## การทำงานของระบบ Offline

### 1. การบันทึกข้อมูล

เมื่ออยู่ใน offline mode:

- ข้อมูลทั้งหมดจะถูกบันทึกใน IndexedDB (ฐานข้อมูลในเบราว์เซอร์)
- แต่ละรายการจะถูกทำเครื่องหมายว่า "ยังไม่ได้ซิงค์" (synced = 0)
- ระบบจะแสดงจำนวนรายการที่รอซิงค์

### 2. Auto Sync เมื่อกลับมา Online

เมื่อมีการเชื่อมต่ออินเทอร์เน็ตอีกครั้ง:

- ระบบจะแสดงข้อความแจ้งเตือน "กลับมาออนไลน์แล้ว"
- Auto sync จะเริ่มทำงานหลัง 1 วินาที
- ข้อมูลจะถูกส่งไปยังเซิร์ฟเวอร์เป็น batch (50 รายการต่อครั้ง)

### 3. Retry Mechanism

หากการ sync ล้มเหลว:

- ระบบจะนับจำนวนครั้งที่พยายาม (retry count)
- จะพยายามส่งอีกครั้งในรอบถัดไป (ทุก 5 นาที)
- รายการที่ล้มเหลวเกิน 3 ครั้งจะแสดงใน "รายการซิงค์ล้มเหลว"

### 4. Service Worker Caching

- Static assets (HTML, CSS, JS) จะถูก cache ไว้
- แอปสามารถโหลดได้แม้ไม่มีอินเทอร์เน็ต
- Cache จะถูกอัพเดทอัตโนมัติเมื่อมีเวอร์ชันใหม่

## การตรวจสอบสถานะ Offline

### ที่ App Bar

- แสดง chip สีเขียว "Online" เมื่อมีอินเทอร์เน็ต
- แสดง chip สีส้ม "Offline" เมื่อไม่มีอินเทอร์เน็ต

### ที่หน้า Settings

- แสดงสถานะการเชื่อมต่อ
- แสดงจำนวนรายการที่รอซิงค์
- ปุ่ม "ซิงค์ข้อมูลตอนนี้" จะถูก disable เมื่อ offline

### ที่หน้า Sync Status

- แสดงสถิติการซิงค์
- แสดงรายการที่ซิงค์ล้มเหลว
- แสดงอัตราความสำเร็จ (%)

## การใช้งานแนะนำ

### สำหรับพื้นที่ที่มีสัญญาณไม่เสถียร

1. **เตรียมข้อมูลก่อนออกสำรวจ**
   - ดึงข้อมูลการลงทะเบียนล่าสุด (เข้าแอปขณะมีอินเทอร์เน็ต)
   - ตรวจสอบว่าข้อมูลอัพเดทแล้ว

2. **ขณะสำรวจ (Offline)**
   - ถ่ายรูปป้ายทะเบียน
   - กรอกข้อมูลด้วยตนเอง
   - ระบบจะบันทึกข้อมูลและ GPS อัตโนมัติ

3. **เมื่อกลับมามีสัญญาณ**
   - เปิดแอป
   - ตรวจสอบที่หน้า "สถานะซิงค์ข้อมูล"
   - กดปุ่ม "ซิงค์ข้อมูลตอนนี้" (หากต้องการซิงค์ทันที)

### การแก้ปัญหา

**ข้อมูลไม่ถูกซิงค์**

1. ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
2. เข้าหน้า "สถานะซิงค์ข้อมูล"
3. กดปุ่ม "ซิงค์ข้อมูลตอนนี้"
4. หากยังไม่สำเร็จ ตรวจสอบ Console สำหรับ error log

**รายการซิงค์ล้มเหลวซ้ำๆ**

1. เข้าหน้า "สถานะซิงค์ข้อมูล"
2. ดูรายการที่ล้มเหลว
3. กดปุ่ม "ลองซิงค์อีกครั้ง"

**แอปไม่โหลด**

1. ล้าง Browser Cache
2. Reload หน้าเว็บ
3. ตรวจสอบว่า Service Worker ทำงานอยู่ (DevTools > Application > Service Workers)

## Technical Details

### Database Structure (IndexedDB)

```javascript
scannedPlates: {
  id: auto-increment,
  plateNumber: string,
  timestamp: Date,
  synced: 0 | 1,
  retryCount: number,
  lastSyncAttempt: Date,
  syncedAt: Date,
  gps: { latitude, longitude, accuracy },
  offline: boolean
}
```

### Sync Strategy

- **Interval**: ทุก 5 นาที (เมื่อ online)
- **Batch Size**: 50 รายการต่อครั้ง
- **Max Retry**: 3 ครั้ง (หลังจากนั้นต้อง manual retry)

### Browser Support

- Chrome/Edge (recommended)
- Safari (iOS 11.3+)
- Firefox

### Storage Limits

- IndexedDB: ไม่จำกัด (ขึ้นกับพื้นที่ว่างในเครื่อง)
- Service Worker Cache: ~50MB (ขึ้นกับเบราว์เซอร์)

## Performance Tips

1. **ลบข้อมูลเก่าเป็นระยะ**: ใช้ฟังก์ชัน "ล้างแคช" เมื่อข้อมูลซิงค์เรียบร้อยแล้ว
2. **Export ข้อมูลสำรอง**: Export ข้อมูลเป็น JSON เป็นระยะ
3. **ตรวจสอบ Storage**: ดูขนาดข้อมูลที่เก็บไว้ (DevTools > Application > Storage)
